1.2 数据流输入（速度快，比赛中常用）

  1. 新建BufferedReader对象（固定格式）；

  2. 使用BufferedReader对象的readLine()方法读取一整行数据，存入String中；

  3. 使用字符串切片方法split(“分割符”)将一整行的数据分成若干串，存入String数组中；

  4. 使用封装数据类的 valueOf(String)方法，将字符串转换为基本数据类型。

1.3 数据流输入配合StreamTokenizer读取（速度快，比赛中常用）

  1. 新建 BufferedReader对象（固定格式）；
  2. 创建 StreamTokenizer()对象，参数为 BufferedReader对象；
  3. 使用 nextToken()方法获取下一组标记，默认是按照空格分割的（回车，tab是结束符）；
  4. 使用 .nval方法解析该组元素，默认返回double值。（ .sval方法解析返回字符串）

缺点： .nval方法只能解析数值型，.sval方法只能解析字母类型，而对于特殊类型（如：**、\ ）

1.4 总结

类型 优点 缺点
BufferedReader对象 可以读取各种数据 操作相对麻烦（切割、转换）
StreamTokenizer对象 操作相对简单 不能读取特殊类型

4.2 计数问题

[P1980 NOIP2013 普及组] 计数问题 - 洛谷 | 计算机科学教育新生态 (luogu.com.cn)

思路一：暴力枚举（比较慢）

将数字转字符串再转为字符数组后逐位比较

思路二：规律解法

①a,b,c为n的三部分，求哪一位x的个数，b就为那一位数，a为b左边的数，c为b右边的数

（如求1~728中十位7的个数，则a=7，b=2，c=8）

②如果b>x，说明有(a+1)m个x
（如求1~728中个位7的个数，则为（72+1）1=73）

③如果b=x，说明有am+c+1个x**（如求1~728中百位7的个数，则为0100+28+1=29）

④如果b<x，说明有am个x

（如求1~728中十位7的个数，则为710个）
注意：x=0的情况和x！=0的情况有所不同

③判断所制造的回文数是否为质数，是 >> 输出，不是 >> 继续
